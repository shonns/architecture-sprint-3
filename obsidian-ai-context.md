# Obsidian Web Clipper + ИИ агент: хранение контекста между сессиями

## Идея

Большинство ИИ агентов (ChatGPT, Claude, Gemini и другие) не помнят предыдущие разговоры между независимыми сессиями. Каждый новый чат начинается «с чистого листа». Связка **Obsidian + Web Clipper** позволяет создать личную базу знаний, в которую автоматически сохраняется контекст ваших диалогов с ИИ, а затем легко передавать нужный контекст агенту в начале новой сессии.

---

## Компоненты системы

| Компонент | Роль |
|---|---|
| **Obsidian** | Локальное хранилище заметок в формате Markdown; вся база знаний находится у вас на диске |
| **Obsidian Web Clipper** | Браузерное расширение для сохранения веб‑страниц (и содержимого чата) в Obsidian одним кликом |
| **ИИ агент** | ChatGPT, Claude, Gemini, локальная модель через Open WebUI и т.д. |
| **Шаблоны Web Clipper** | Задают, *какие* части страницы сохранять и в каком виде |

## Шаг 1. Установка и базовая настройка

### 1.1 Установите Obsidian

Скачайте Obsidian с официального сайта <https://obsidian.md> и создайте новое хранилище (vault), например `AI-Context`.

Рекомендуемая структура папок:

```
AI-Context/
├── sessions/          # отдельный файл на каждый диалог
├── projects/          # контекст по конкретным проектам
├── templates/         # шаблоны заметок
└── index.md           # оглавление / дашборд
```

### 1.2 Установите Obsidian Web Clipper

1. Откройте официальную страницу расширения:
   - Chrome / Edge: <https://chromewebstore.google.com/detail/obsidian-web-clipper/cnjifjpddelmedmihgijeibhnjfabmlf>
   - Firefox: <https://addons.mozilla.org/firefox/addon/web-clipper-obsidian/>
2. Установите расширение и нажмите на иконку в браузере.
3. Укажите **имя хранилища** (vault name) — оно должно совпадать с именем вашего vault в Obsidian.
4. Включите **Obsidian URI** в настройках Obsidian: `Settings → General → Advanced → Enable Obsidian URI`.

## Шаг 2. Настройка шаблона для сохранения диалогов с ИИ

Web Clipper поддерживает кастомные шаблоны на базе [Handlebars](https://handlebarsjs.com/). Создайте шаблон специально для страниц чата.

### Пример шаблона `AI Session`

В настройках Web Clipper (`Options → Templates → Add template`) создайте шаблон:

**Имя шаблона:** `AI Session`

**URL‑фильтр** (применять только на нужных сайтах):
```
chatgpt.com, claude.ai, gemini.google.com, chat.mistral.ai
```

**Папка назначения:**
```
sessions/{{date:YYYY-MM-DD}}
```

**Имя файла:**
```
{{date:YYYY-MM-DD HH-mm}} {{title}}
```

**Содержимое заметки (шаблон):**
```markdown
---
date: {{date:YYYY-MM-DD HH:mm}}
source: {{url}}
tags: [ai-session, {{#if author}}{{author}}{{/if}}]
project: ""
status: raw
---

# {{title}}

> Сохранено: {{date:YYYY-MM-DD HH:mm}}
> Источник: {{url}}

## Контекст сессии

{{content}}

---

## Ключевые выводы

- 

## Следующие шаги

- 

## Связанные заметки

- 
```

> **Совет.** Поле `content` захватывает выделенный текст на странице. Чтобы сохранить конкретный фрагмент диалога — сначала выделите его, затем нажмите горячую клавишу Web Clipper (по умолчанию `Alt+Shift+O`).

## Шаг 3. Рабочий процесс сохранения контекста

```
Диалог с ИИ
    │
    │  1. Выделяете важные части разговора
    │     (итоги, решения, код, список задач)
    │
    ▼
Web Clipper (Alt+Shift+O)
    │
    │  2. Расширение формирует заметку
    │     по шаблону и сохраняет в Obsidian
    │
    ▼
Obsidian Vault (sessions/)
    │
    │  3. Добавляете тег проекта,
    │     дополняете «Ключевые выводы»
    │
    ▼
Готовый контекст для следующей сессии
```

### Что стоит сохранять

- Итоговый ответ или решение задачи
- Архитектурные решения и их обоснование
- Сгенерированный код с описанием
- Открытые вопросы / TODO, которые нужно продолжить
- Параметры и ограничения, которые агент должен помнить

## Шаг 4. Передача контекста в новую сессию

Когда вы начинаете новый разговор с ИИ, вставьте в первое сообщение контекст из Obsidian.

### Вариант A — вручную через поиск Obsidian

1. Откройте Quick Switcher в Obsidian (`Ctrl+O`).
2. Найдите нужную заметку по дате или тегу.
3. Скопируйте раздел «Ключевые выводы» и «Следующие шаги».
4. Вставьте в начало нового чата с ИИ.

### Вариант B — шаблон «системного промпта»

Создайте в Obsidian файл `templates/context-prompt.md`:

```markdown
## Контекст предыдущих сессий

Ниже — сводка из предыдущих диалогов по проекту **{{project}}**.

### Принятые решения
{{decisions}}

### Текущее состояние
{{current_state}}

### Ограничения и требования
{{constraints}}

### Открытые вопросы
{{open_questions}}

---
Продолжи работу, опираясь на этот контекст.
```

Заполняйте шаблон перед каждой сессией и вставляйте его как первое сообщение агенту.

### Вариант C — плагин Obsidian для автоматической сборки контекста

Установите плагин **Dataview** (`Settings → Community Plugins → Dataview`). Создайте в `index.md` динамический запрос, который собирает все заметки по проекту:

````markdown
## Последние сессии по проекту

```dataview
TABLE date, status FROM "sessions"
WHERE contains(tags, "ai-session") AND project = "my-project"
SORT date DESC
LIMIT 10
```
````

## Шаг 5. Дополнительные инструменты для автоматизации

### 5.1 Плагин «Templater»

Плагин **Templater** позволяет запускать JavaScript внутри шаблонов Obsidian. Например, автоматически добавлять дату, имя проекта из текущей папки или форматировать текст.

### 5.2 Локальный ИИ через Open WebUI

Если вы используете локальную языковую модель (Ollama + Open WebUI), можно настроить экспорт истории чата в папку, которую Obsidian отслеживает как vault. Тогда заметки будут появляться автоматически без ручного клиппинга.

### 5.3 Интеграция через Zapier / Make

Если ИИ агент имеет API или поддерживает webhooks (например, n8n, Flowise, CrewAI), можно автоматически записывать итоги сессий прямо в файловую систему vault и синхронизировать через облако (iCloud, Obsidian Sync, Git).

## Рекомендуемая структура заметки-контекста

```markdown
---
date: 2024-11-15 14:30
project: architecture-sprint-3
tags: [ai-session, claude, microservices]
status: reviewed          # raw | reviewed | archived
---

# Сессия: Проектирование микросервисной архитектуры

## Задача сессии
Спроектировать разбиение монолита умного дома на микросервисы.

## Ключевые решения
- Выделены сервисы: heating-service, telemetry-service, user-service
- Взаимодействие через Kafka (async) и REST (sync для критичных операций)
- База данных — отдельная на каждый сервис (паттерн Database per Service)

## Открытые вопросы
- [ ] Как организовать distributed transactions между сервисами?
- [ ] Нужен ли API Gateway или достаточно Ingress в k8s?

## Следующие шаги
1. Нарисовать диаграмму C4 Containers
2. Описать контракты API между сервисами
```
